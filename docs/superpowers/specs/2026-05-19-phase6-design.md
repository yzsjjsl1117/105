## 1. 产品详情页 UI 重构

### 设计方向

**自然茶境** — 山林意境、暖色调、质感纹理，像打开茶叶包装的体验。

### 色调

| 角色 | 色值 | 用途 |
|------|------|------|
| 主色 | `#1a3a2a` | 品牌绿，按钮、强调文字 |
| 强调色 | `#8B6914` / `#C4953A` | 茶金，分隔线、装饰元素 |
| 底色 | `#FDF8F0` | 米白，页面背景 |
| 文字 | `#333` / `#555` / `#888` | 标题/正文/辅助 |

### 关键变更

| 项目 | 当前 | 改为 |
|------|------|------|
| 产品特性 | 4 个卡片（icon + 标题 + 描述） | 4 张纵向排列的大图，顺序：产地 → 外形 → 香气 → 汤色 |
| 冲泡方法 | 4 步卡片（数字 + 标题 + 内容） | 1 张宣传大图 + 文字叠加 |
| 储存方法 | 左侧竖条卡片 | 1 张宣传大图 + 文字叠加 |
| 技术方式 | Tailwind className + inline style 混用 | 统一用内联 style（与项目其他页面一致） |

### 图片说明

产品特性、冲泡方法、储存方法的图片由产品方提供，图片内包含解释文字。前端只负责图片排列和响应式适配，不负责图片内的文字排版。

## 2. 响应式适配

### 策略：统一断点 + 按页面类型适配

**核心断点：** `768px`（移动端 / 桌面端分界）

| 页面类型 | 桌面端 | 移动端（≤768px） |
|---------|--------|-----------------|
| 产品详情页 | 左右分栏 | 上图下文，单栏纵向滚动 |
| 购物车 | 表格式（表头+行） | 卡片式（每个商品一张卡片） |
| 结算页 | 宽表单 | 全宽表单 |
| 支付页 | 居中卡片 | 全宽卡片 |
| 个人中心 | 左侧导航 + 右侧内容 | 顶部分类横滑 + 下方内容 |
| 后台管理 | 表格 + 侧边导航 | 表格横滑，顶栏收缩 |
| 首页（品牌故事等） | 已有基础响应式 | 补充 Navbar/Footer |

### 通用规则

- **容器宽度**：`max-width` 从固定 px 改为百分比 + 最大宽度约束
- **导航栏**：桌面端 64px/44px 内边距 → 移动端 16px
- **表单输入框**：桌面端固定 320px → 移动端 `width: 100%`
- **表格**：移动端 `overflow-x: auto` 支持横滑
- **网格**：多列网格在移动端改为单列
- **字号**：移动端标题和价格适当缩小

### 影响范围

涉及全部页面路由（约 15 个文件），优先处理用户端页面（前台），后台管理可以更宽松。

## 3. SEO 优化

### 全站 metadata

为每个页面路由添加 `metadata` 或 `generateMetadata`：

| 页面 | title | description |
|------|-------|-------------|
| `/` | 瀹岭 — 云雾之间的馈赠 | 已有 |
| `/products/[slug]` | `{产品名} - 瀹岭` | 已有（产品描述） |
| `/cart` | 购物车 - 瀹岭 | 已有（需补充） |
| `/checkout` | 确认下单 - 瀹岭 | 已有（需补充） |
| `/payment/[orderId]` | 确认支付 - 瀹岭 | 已有（需补充） |
| `/payment/success` | 支付成功 - 瀹岭 | 已有（需补充） |
| `/orders/[id]` | 订单详情 - 瀹岭 | 已有（需补充） |
| `/auth/login` | 登录 - 瀹岭 | 已有（需补充） |
| `/auth/register` | 注册 - 瀹岭 | 已有（需补充） |
| `/account` | 个人中心 - 瀹岭 | 已有（需补充） |

### Open Graph

根布局添加 `og:image`、`og:type`、`twitter:card`：

```typescript
openGraph: {
  title: "瀹岭 — 云雾之间的馈赠",
  description: "...",
  images: ["/images/og-image.png"],  // 需准备一张 1200×630 的品牌图
}
```

### sitemap.xml + robots.txt

- `sitemap.xml`：通过 Next.js `app/sitemap.ts` 自动生成，包含首页 + 产品详情页 + 静态页
- `robots.txt`：通过 Next.js `app/robots.ts` 生成

### 技术说明

- 后台页面（`/admin/*`）不需要 SEO
- 所有页面使用 `layout.tsx` 中的 `metadata` 模板进行复用

## 4. middleware → proxy 迁移

Next.js 16 弃用 `middleware.ts`，迁移到 `proxy.ts`：

- 文件名：`src/middleware.ts` → `src/proxy.ts`
- 导出名：`export default function middleware` → `export default function proxy`
- 功能逻辑不变
- matcher 配置保持不变

风险极低，纯重命名 + API 适配。

## 5. Vercel 部署

### 前置条件

- GitHub 仓库已推送
- 环境变量齐全：

| 变量 | 来源 |
|------|------|
| `DATABASE_URL` | Supabase 连接串 |
| `AUTH_SECRET` | NextAuth 密钥 |
| `SUPABASE_URL` | Supabase 项目 URL |
| `SUPABASE_SERVICE_KEY` | Supabase service_role key |

### 步骤

1. 在 Vercel 中 import GitHub 仓库
2. 配置环境变量
3. 部署（Vercel 自动检测 Next.js 项目）
4. 如需自定义域名，在 Vercel Domains 中添加

### 注意事项

- Supabase 免费层数据库在 7 天无活动后可能暂停，部署后需定期访问
- `prisma generate` 在 Vercel 构建时会自动执行（通过 `postinstall` 脚本）
- 中间件迁移到 `proxy.ts` 在部署前完成
